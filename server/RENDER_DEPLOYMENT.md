# 🚀 AgriVision Backend - Render Deployment Guide

## Overview
This guide will help you deploy the AgriVision backend API to Render.com, a cloud platform that makes deployment simple and reliable.

## Prerequisites
- A [Render.com](https://render.com) account (free tier available)
- Git repository with your backend code
- MongoDB Atlas database (already configured)
- AWS S3 bucket for file uploads (already configured)
- Google Gemini API key (already configured)

## 📋 Pre-Deployment Checklist

### 1. Environment Variables Required
Before deploying, ensure you have these values ready:

| Variable | Description | Source |
|----------|-------------|--------|
| `MONGO_URI` | MongoDB connection string | Your MongoDB Atlas cluster |
| `JWT_SECRET` | Secret key for JWT tokens | Auto-generated by Render |
| `GEMINI_API_KEY` | Google Gemini AI API key | Google AI Studio |
| `AWS_ACCESS_KEY_ID` | AWS access key | AWS IAM Console |
| `AWS_SECRET_ACCESS_KEY` | AWS secret key | AWS IAM Console |
| `AWS_S3_BUCKET_NAME` | S3 bucket name | AWS S3 Console |
| `CLIENT_URL` | Frontend application URL | Your frontend deployment URL |

### 2. Current Configuration Status
✅ **Already Configured:**
- MongoDB URI (from `.env`)
- AWS S3 credentials (from `.env`)
- Gemini API Key (from `.env`)
- Node.js version (>=18.0.0)
- Build and start scripts

⚠️ **Needs Update After Deployment:**
- `CLIENT_URL` - Update with your actual frontend URL

## 🎯 Deployment Steps

### Method 1: Deploy via Render Dashboard (Recommended)

#### Step 1: Push Code to Git Repository
```bash
cd d:\MASAI\AgriVision\server
git init
git add .
git commit -m "Initial commit with Render configuration"
# Add your remote repository
git remote add origin <your-git-repo-url>
git push -u origin main
```

#### Step 2: Create New Web Service on Render
1. Go to [Render Dashboard](https://dashboard.render.com/)
2. Click **"New +"** → **"Web Service"**
3. Connect your Git repository
4. Configure the service:

   **Basic Settings:**
   - **Name**: `agrivision-backend` (or your preferred name)
   - **Region**: Oregon (or closest to your users)
   - **Branch**: `main` (or your deployment branch)
   - **Root Directory**: Leave blank (or `AgriVision/server` if repo root is different)
   - **Runtime**: `Node`
   - **Build Command**: `npm install && npm run build`
   - **Start Command**: `npm start`

   **Instance Type:**
   - Select **Free** plan (for testing) or **Starter** ($7/month for production)

### Method 2: Deploy via render.yaml (Blueprint)

If you want to use the `render.yaml` file:

## 🔧 Post-Deployment Configuration

### 1. Update CORS Settings
After getting your Render URL, update the `CLIENT_URL` environment variable:

```
CLIENT_URL=https://your-frontend-domain.com
```

### 2. Test API Endpoints
Test the following endpoints:

```bash
# Health Check
curl https://your-app.onrender.com/api/health

# Home Page (API Documentation)
curl https://your-app.onrender.com/

# Authentication
curl -X POST https://your-app.onrender.com/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{"name":"Test User","email":"test@example.com","password":"password123"}'
```

### 3. Configure Frontend
Update your frontend's API base URL to point to your Render deployment:

```typescript
// In your frontend .env.local
NEXT_PUBLIC_API_URL=https://agrivision-backend.onrender.com/api
```

## 📊 Monitoring & Logs

### View Logs
- Go to Render Dashboard → Your Service → **Logs** tab

- Monitor CPU, Memory, and Request counts

### Health Checks
- Render automatically checks `/api/health` endpoint
- If health check fails, Render will restart your service

## ⚠️ Important Notes

### Free Tier Limitations
- **Spin Down**: Free services spin down after 15 minutes of inactivity
- **Cold Start**: First request after spin-down takes 30-60 seconds
- **Solution**: Upgrade to Starter plan ($7/month) for always-on service

### Database Connection
- MongoDB Atlas connection is already configured
- Ensure your MongoDB Atlas IP whitelist includes `0.0.0.0/0` (all IPs) for Render access

### File Uploads
- Files are uploaded to AWS S3 (not stored on Render)
- Ensure S3 bucket CORS is configured correctly

### Rate Limiting
- Current config: 100 requests per 15 minutes per IP
- Adjust `RATE_LIMIT_MAX` based on your needs

## 🔐 Security Best Practices

1. **Never commit `.env` files** to Git (already in `.gitignore`)
2. **Use Render's environment variables** for all secrets
3. **Rotate JWT_SECRET** periodically
4. **Enable HTTPS** (automatic on Render)
5. **Set strong CORS policies** with specific CLIENT_URL
6. **Monitor logs** for suspicious activity

## 📈 Scaling Considerations

When your app grows:

1. **Upgrade Plan**: Free → Starter ($7/mo) → Standard ($25/mo)
2. **Add Regions**: Deploy closer to users
3. **Database**: Upgrade MongoDB Atlas plan
4. **CDN**: Add Cloudflare for static assets
5. **Monitoring**: Integrate Sentry or similar tools
